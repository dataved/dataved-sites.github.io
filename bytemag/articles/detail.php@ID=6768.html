Анализ данных и корпоративная отчетность в Cognos 8 BI

15.02.2006

Александр Карасёв, Борис Толмач, Тамара Прийменко
консультанты практики информационных аналитических систем, Columbus IT Partner Russia

Компания Cognos (http://www.cognos.com), работающая на ИТ-рынке с 1969 г., специализируется на разработке программных решений для управления эффективностью предприятия (Corporate Performance Management, CPM) и предлагает систему, призванную охватить полный цикл управления предприятием на основе единой технологической платформы (рис. 1). Функционально процесс управления предприятием рассматривается с трех позиций:

  • анализ данных, позволяющий оценить тенденции развития;
  • мониторинг текущей деятельности, позволяющий ускорить принятие решений;
  • планирование, задающее ориентиры развития.

Рис. 1. Управление эффективностью в решениях компании Cognos.

Вместе эти три области помогают получить продуманный и аргументированный ответ на три основных вопроса, с которыми сталкиваются все сотрудники, вовлеченные в процесс принятия решений: "Каково состояние бизнеса сейчас?", "Как идет его развитие?", "Что следует сделать в будущем?".

В сентябре 2005 г. компания Cognos представила новую версию программного комплекса, предназначенного для мониторинга показателей эффективности, предоставления отчетности и анализа данных масштаба предприятия, - Cognos 8 Business Intelligence (BI). Этот пакет позволяет проводить анализ на основе реляционных, OLAP и других источников данных и выстраивать системы отчетности с развитыми средствами интеграции и визуализации данных.

В основе Cognos 8 BI лежит концепция единого портала для доступа ко всем возможностям системы, реализованной на базе Web-технологий, что не только обеспечивает высокую степень масштабирования решения, но и значительно сокращает затраты на развертывание и обслуживание системы.

В состав Cognos 8 BI входят следующие компоненты:

  • Cognos Connection - Интернет-портал для предоставления единого доступа ко всем компонентам системы;
  • Cognos Query - конструктор отчетов для начинающих пользователей, позволяющий строить сводные отчеты с помощью визуальных методов;
  • Cognos Report Studio - инструмент создания профессиональных многостраничных отчетов с произвольной структурой представления информации;
  • Cognos Analysis Studio - инструмент анализа данных, представленных в различных OLAP-системах;
  • Cognos Metrics Manager - инструмент предоставления оперативной информации о показателях эффективности деятельности (KPI) и построения на их основе взаимосвязанной системы показателей эффективности (Balanced Scorecard);
  • Cognos Event Studio - система контроля значений выбранных пользователем показателей, с возможностями автоматической рассылки уведомлений;
  • Cognos BI OLAP Modeling - компонент для создания OLAP-источников данных;
  • Cognos Framework Manager - инструмент объединения источников информации и построения на их основе "метамодели", предоставляемой пользователю;
  • Cognos Decision Stream - инструмент построения алгоритмов извлечения и преобразования информации, предназначенный в первую очередь для построения хранилищ данных (ETL-средство).

Система предлагает широкие возможности для комфортной работы в самых различных инфраструктурах. Cognos 8 BI - это кроссплатформенное и многокомпонентное решение, поддерживающее платформы Windows и Unix. Все компоненты системы объединены общим порталом, обеспечивающим единую модель безопасности и возможность свободного взаимодействия отдельных компонентов между собой. Все компоненты, предназначенные для работы с отчетностью, анализа данных, управления пользователями и компонентами системы, доступны с помощью обычного Web-браузера, не требуя дополнительного программного кода на стороне клиента. Благодаря этому пользователи могут не заботиться о поддерживаемых платформах, программном обеспечении и своем местоположении. Система также поддерживает мультиязычность, когда не только интерфейс, но и содержимое отчетов предоставляется с учетом пожеланий пользователя.

Интеграция различных источников данных

Будучи системой масштаба предприятия, Cognos 8 BI поддерживает широкий спектр источников данных, а именно:

  • реляционные (IBM, Oracle, SQL, Teradata, Sybase, ODBC и т. д.);
  • на основе Интернет-технологий (XML, JDBC, LDAP, WSDL);
  • хранилища данных OLAP (Cognos PowerCubes, Cognos Planning Contributor Cubes и Cognos Finance Cubes, Microsoft OLAP, IBM DB2/OLAP, Hyperion Essbase, SAP Business Warehouse (BW), ROLAP).

Расширенная возможность сопоставления данных из разных источников для одного отчета - одно из наиболее важных нововведений Cognos. Теперь разработчики отчетов могут комбинировать и связывать данные из произвольного набора источников (реляционная база данных, XML, Excel, OLAP и другие источники) в едином представлении, доступном для пользователей. Такая связь может существовать как в режиме реального времени, так и на основе создания хранилищ или витрин данных с помощью предлагаемых механизмов извлечения и преобразования информации (ETL-средства).

Средства интеграции данных

Доступ пользователей к информации осуществляется на основе метамодели, преобразующей физическую структуру источников данных в группированный набор, представленный в терминах и наименованиях, применяемых в конкретной организации.

С помощью инструмента Framework Manager администратор системы задает необходимые соединения, вычисления, настраивает параметры безопасности для исходных данных (рис. 2). В процессе формирования логики связей Framework Manager способен распознавать и использовать информацию о связях между таблицами, заданную в процессе проектирования баз данных. Количество источников информации может быть произвольным - в рамках одной метамодели можно обеспечить, например, соединение данных из баз данных Microsoft SQL и Oracle. В дальнейшем при доступе к данным такая связь будет создаваться и поддерживаться автоматически.

Рис. 2. Framework Manager - средство для интеграции данных.

На следующем этапе администратор создает бизнес-ориентированную модель, связанную с определенным направлением работы организации (например, работа с заказами). Показатели в данной модели обозначаются и группируются в наборы в соответствии с их логическим значением, вне зависимости от их физического наименования и расположения в источниках данных. Связи, необходимые для группировки показателей, формируются системой автоматически на основании информации, определенной на первом этапе. Такая метамодель, описанная в терминах и наименованиях, применяемых в конкретной организации, и предоставляется пользователям.

Далее в процессе работы отчеты, созданные пользователями, транслируются в соответствующие запросы к базам данных. Cognos 8 BI имеет собственный механизм оптимизации запросов, позволяющий определить стратегию обработки данных: на основе только возможностей источников данных или с применением локальной обработки. В большинстве случаев система сама выбирает стратегию обработки, но профессиональные авторы отчетов могут скорректировать алгоритм.

Отраслевые решения для Oracle и SAP

В целях ускорения внедрения своих программных продуктов компания Cognos разработала пакеты с преднастроенными комплексами аналитики и отчетности для систем Oracle E-Business Suite и SAP R/3. Такие пакеты предоставляют возможности для анализа в следующих областях: взаимодействие с клиентами и поставщиками, логистика и управление складом, управление финансами, решения для розницы. Общее количество преднастроенных отчетов составляет более 100 позиций для каждой из систем. В зависимости от сложности учетной системы обеспечивается или непосредственный доступ к базам данных, или построение хранилища данных средствами Cognos BI. При этом сами хранилища разработаны на высоком профессиональном уровне, поддерживается инкрементальное обновление и корректная работа с измерениями.

Нововведения в Cognos 8 BI

Корпоративный портал

Корпоративный портал системы обеспечивает единый для всех пользователей доступ к данным, позволяя не только группировать предоставляемую информацию согласно бизнес-требованиям, но и создавать свои собственные информационные панели. Пользователям выделяется собственное именное пространство, где они могут реализовать альтернативную иерархию отчетов и назначать их получение по расписанию, независимо от административных настроек. В рамках данного портала также обеспечиваются административные инструменты управления политикой безопасности и элементами системы.

Среди нововведений портала следует отметить, во-первых, возможность обмена между сотрудниками компании заранее подготовленными наборами отчетов и аналитик через портал вместо традиционных распечаток, а во-вторых, расширенную поддержку сторонних информационных порталов - Plumtree, порталы в IBM WebSphere and SAP Netweaver, а также технологий Web Services for Remote Portlets (WSRP).

Система оперативной отчетности

Система оперативной отчетности представлена в Cognos 8 BI двумя инструментами - Query Studio и Report Studio.

Query Studio - инструмент, дающий пользователям возможность с минимальным уровнем знаний получить необходимый отчет "на лету", используя методы визуального конструктора (рис. 3). Полученный таким образом отчет можно сохранить для последующего использования, распечатать, добавить форматирование и графики. С помощью этого инструмента можно формировать сводные таблицы и плоские отчеты, не прибегая к помощи ИТ-специалистов.

Рис. 3. Query Studio предоставляет визуальные методы создания отчетов для непрофессиональных пользователей.

Новые возможности Cognos 8 Query Studio включают:

  • механизм drill-down (углубление по иерархии) как для таблиц, так и для графиков;
  • условное форматирование - способы придать отчету более информативный вид за счет выделения значений цветом или шрифтом;
  • поддержку фильтров данных и математических вычислений по столбцам или строкам, не включенным в поля отчета.

Report Studio - инструмент создания профессиональных отчетов, позволяющий в произвольной форме комбинировать данные из различных источников (рис. 4). В арсенале данного инструмента присутствует следующая функциональность:

  • создание многостраничных отчетов с произвольным расположением таблиц и графиков;
  • полный контроль над свойствами элементов отчета;
  • возможность получать и объединять данные из нескольких метамоделей в рамках одного отчета;
  • внедрение мультимедийных объектов;
  • условные вычисления на основе данных метамодели или полученного отчета;
  • расширенные возможности условного форматирования - возможность персонализации вида отчета (скрытие строк или столбцов, цветовое кодирование значений) на основе неограниченного числа критериев;
  • задание параметров и внедрение элементов управления фильтрацией данных в тело отчета.

Рис. 4. Пользовательский интерфейс Report Studio предоставляет полный контроль над каждым элементом данных, включенных в отчет.

В версии Cognos 8 в Report Studio внесено довольно много усовершенствований. Наиболее многообещающее нововведение в ее функционале - поддержка OLAP-источников данных. Возможности представления данных в Report Studio позволяют обойти ограничения, традиционно присущие функционалу OLAP-клиентов, и получить отчет с произвольной структурой.

Кроме того, появились возможности условного отображения столбцов в отчете в зависимости от полученных данных или пользовательских настроек и поддержка персональных стилей для шрифтов, цветовых схем, стандартного форматирования страниц, что позволяет ускорить разработку отчетов в едином стиле компании.

Создание информационных панелей - новая функциональность, позволяющая объединить на одном экране информацию из разнообразных источников, эффективно представляя ее в графическом виде. Новые типы графиков, поддержка географических карт и интерактивных объектов позволяют легко воспринимать большие объемы информации.

Анализ данных

В интегрированной системе Cognos 8 на замену Cognos PowerPlay, инструменту анализа данных из предыдущей версии Cognos 7 BI, пришел новый компонент - Analysis Studio (рис. 5). Он предназначен для анализа данных, сформированных с применением OLAP-технологий.

Рис. 5. Analysis Studio предлагает богатые графические возможности представления анализируемой информации.

Вобрав в себя все лучшие идеи из PowerPlay, Analysis Studio предлагает в первую очередь улучшенный пользовательский интерфейс. Все возможности реализованы в нем с помощью дружественного визуального Web-интерфейса, не требующего знаний языков программирования.

В Analysis Studio также существенно расширены возможности проведения анализа. В числе нововведений:

  • поддержка прямого доступа к большому количеству OLAP-платформ: Cognos PowerCubes, Cognos Planning Contributor Cubes and Cognos Finance Cubes, Microsoft OLAP, IBM DB2/OLAP, Hyperion Essbase;
  • полная интеграция с другими инструментами Cognos 8 BI (возможность доработки и использования в Report Studio отчетов, полученных в Analysis Studio, передача данных и управления в другие компоненты системы, внесение подготовленных отчетов в персональные страницы);
  • поддержка ROLAP-баз данных;
  • расширенные возможности работы с измерениями (поддержка условного отображения содержимого измерений по различным критериям, например, независимые Top 10 позиций в каждой подгруппе для многоуровневых отчетов);
  • фильтрация и сортировка данных и измерений по условиям "И" и "ИЛИ";
  • создание, сохранение и повторное использование настраиваемых подмножеств, позволяющих комбинировать данные из произвольных измерений, лежащих на различных уровнях иерархии, - это дает возможность проводить анализ данных, не ограничиваясь заданной структурой измерений;
  • улучшенный экспорт отчетов в Excel - поддержка форматирования данных и графического представления информации.

Контроль показателей эффективности

Инструмент Cognos Metrics Manager предназначен для ежедневного контроля показателей эффективности деятельности - как всей компании, так и отдельных ее подразделений и сотрудников (рис. 6). С его помощью пользователи становятся самостоятельными и могут легко видеть, как развивается состояние дел в компании в зависимости от ее стратегии. Они могут устанавливать приоритеты для собственных действий, анализировать влияние своих решений на эффективность компании.

Рис. 6. MetricsManager предоставляет возможности оперативного контроля ситуации.

В общую картину управления бизнесом могут быть включены ключевые показатели эффективности (KPIs), такие, как удовлетворенность персонала, надежность поставщиков и рентабельность клиентов. Компания может сопоставить планируемую и действительную эффективность и предоставить ориентированные на цель плановые метрики тысячам сотрудников по всему предприятию.

Гибкая структура программы позволяет создавать сложные взаимосвязи между разнородными показателями. Процесс создания системы показателей может основываться на любых стандартных или собственных управленческих методах. Cognos Metrics Manager способен отображать итоговые отчеты, исходя из методик Balanced Scorecard или других методов. Из итогового отчета пользователь может легко перейти к детальной информации и проследить историю изменения показателей.

Cognos Metrics Manager способен импортировать данные из любого источника. Данные для заполнения и поддержки карт показателей могут быть получены из источников данных любого поставщика, файлов Excel, текстовых файлов, значений, введенных пользователем, и источников данных Cognos BI. Для автоматизации загрузки данных из текстовых файлов предоставляется утилита Data Loader. Программа сертифицирована ассоциацией Balanced Scorecard Collaborative и включает стратегические карты (Strategy Maps) и другие эффективные карты методологии Kaplan & Norton Balanced Scorecard, которые можно быстро и легко смоделировать при помощи административного меню.

В новой версии реализована полная интеграция Metrics Manager с остальными инструментами Cognos 8: теперь он стал функциональной частью системы Cognos 8 BI, что обеспечивает единую систему безопасности и возможность использования отчетов Cognos 8 Report Studio и Analysis Studio в качестве источников исходных данных для Metrics Manager.

Кроме того, в Cognos 8 Metrics Manager появились следующие усовершенствования:

  • гибкий пользовательский интерфейс - пользователи, обладающие соответствующими правами, могут редактировать группировки, вычисления и свойства метрик, не покидая своего рабочего окружения;
  • расширенные возможности создания диаграмм влияния - теперь пользователи могут в интерфейсе Metrics Manager создавать диаграммы, используя наборы графических примитивов;
  • возможность работать с несколькими базами данных метрик из интерфейса пользователя;
  • автоматическое фокусирование на последних введенных в систему данных - пользователь всегда видит последние введенные данные, даже если они введены в разное время;
  • создание произвольных календарей с нелимитированной группировкой периодов - позволяет идеально подстраиваться под производственный и финансовый календари компании.

Отслеживание состояния показателей

Новый компонент системы - Cognos Event Studio - отслеживает состояние критических показателей и своевременно информирует пользователей при достижении этими показателями пороговых значений.

Отслеживание можно определить для произвольных показателей метамодели; уведомления о наступлении события рассылаются по электронной почте или с помощью добавления информационных строк в портал. Вместе с сообщением возможна доставка отчета с более детальными данными. Дополнительно поддерживается запуск внешних обработчиков для событий, не требующих участия пользователей системы. Ведется журнал истории обработанных событий, поддерживается возможность внесения пользовательских комментариев.

Резюме

Подводя итоги, можно сказать, что новая версия Cognos 8 BI - это очередной логический этап развития линейки продуктов Cognos. Актуальные задачи, такие, как локализация и стандартизация продукта, стоящие перед компанией Cognos в связи с освоением новых рынков (в том числе и российского), в новой версии решаются последовательно и основательно. Наличие в Cognos 8 BI мощного клиента выгодно отличает его от конкурирующих решений и способствует продвижению BI на все рабочие места, где требуется анализ, что влечет за собой повышение эффективности управления на местах и позволяет получить максимальную отдачу от внедрения продуктов BI.

Использование передовых технологий и кроссплатформенность, простота развертывания и обслуживания продуктов Cognos, легкость интеграции с другими информационными системами, удобный, интуитивно понятный и одновременно очень гибкий пользовательский интерфейс - все это не только делает Cognos 8 BI достойной заменой предыдущей версии, но и позволяет ей рассчитывать занять серьезные позиции на российском рынке CPM.
