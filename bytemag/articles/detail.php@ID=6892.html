Средства Microsoft для управления эффективностью бизнеса

Автор: Андрей Колесов
17.09.2007

Корпорация Microsoft (http://www.microsoft.com) уже давно вошла в число ведущих мировых поставщиков программных средств бизнес-аналитики (Business Intelligence, BI). Однако довольно долгое время BI-платформа Microsoft строилась исключительно на базе ее СУБД SQL Server и одного из самых популярных в мире BI-клиентов — приложения Excel. Нужно еще сказать, что свои основные очертания эта BI-система приобрела только в начале 2004 г. с выпуском компонента Microsoft SQL Server Reporting Services*, а законченный интегрированный вид — с появлением в конце 2005 г. Microsoft SQL Server 2005**.

* См. "Средства создания отчетов от Microsoft", «BYTE/Россия» № 3’2004.
** См. "Средства бизнес-аналитики в Microsoft SQL Server 2005", «BYTE/Россия» № 2’2006.

Фактически именно с этого начался новый этап развития BI-платформы Microsoft — создание на фундаменте SQL Server + Excel специализированной настройки, которая была ориентирована на новый для корпорации (впрочем, во многом и для всего ИТ-рынка) класс задач, связанный c управлением эффективностью бизнеса (Business Performance Management, BPM). Первым шагом в этом направлении стал выпуск в ноябре 2005 г. качественно нового члена семейства Microsoft Office — Business Scorecard Manager (BSM) 2005, предназначенного для управления ключевыми бизнес-показателями предприятий.

Второй шаг Microsoft сделала через год, купив компанию ProClarity (с которой ранее очень тесно сотрудничала в сфере BI) вместе с ее продуктом ProClarity Analytics Server (PAS) 6, интегрированным с BI-платформой Microsoft. Почти сразу после этого корпорация объявила о намерении создать в составе семейства Microsoft Office новое серверное решение PerformancePoint Server (PPS), в котором предполагалось объединить и расширить функциональность BSM 2005 и PAS 6, а также включить в него новые средства, разрабатываемые в рамках внутреннего проекта Biz# (BizSharp). Первая предварительная бета-версия PPS 2007 была анонсирована в декабре 2006 г., более подробно она была представлена в мае нынешнего года в Сиэтле на первой конференции Microsoft по тематике BI. Окончательный вариант продукта должен появиться на рынке в ноябре.

Создание BI-платформы подразумевает ее тесную интеграцию со всем офисным семейством Microsoft в целом, и ключевая роль тут отводится Office SharePoint Server (SPS) 2007 как связующей базовой платформе поддержки групповой работы и управления контентом. Отметим также, что в составе Office 2007 появился новый серверный компонент на базе SPS 2007 — Excel Services с возможностями публикации результатов анализа, полученных в Excel.

С учетом всего сказанного BI-платформу Microsoft можно представить в виде системы различных серверов, приложений и компонентов, архитектура которой представлена на рис. 1.

Рис. 1. Архитектура платформы Microsoft Business Intelligence.

Бизнес-аналитика в системе управления предприятием

Сегодня на многих предприятиях используются разнородные, порой плохо взаимодействующие друг с другом программные комплексы и источники данных. Это объясняется целым рядом как субъективных (решение задач по мере их появления, особенности корпоративной культуры), так и объективных причин (выбор оптимального решения для конкретной задачи, соображения стоимости и т. д.).

В то же время управление современным бизнесом требует, чтобы все сотрудники предприятия всегда имели доступ к любой информации, нужной им для принятия решений — от частных до стратегических, и получали ее в нужное время и в нужном формате. Реализация этой идеи оказывается не слишком простой не только из-за неоднородности информационных систем, но и по причине технических трудностей при одновременном выполнении оперативных и аналитических задач. Например, попытки составления отчетов средствами оперативных систем серьезно снижают их быстродействие, к тому же возникают проблемы с использованием актуальной информации о текущих операциях при выполнении углубленного анализа данных.

В комплексе BI-средств Microsoft эти проблемы разрешаются за счет разделения на техническом уровне процессов функционирования оперативных систем и средств поддержки принятия решений; при этом обеспечивается прямое подключение ко всем возможным источникам данных. Доступ к необходимой информации каждый сотрудник получает в удобном для себя виде, используя для этого, например, хорошо знакомые офисные приложения. В целом в BI-системе имеется три уровня, каждый из которых соответствует типу решаемых задач: хранилище данных, отчетность и анализ, управление эффективностью бизнеса.

Как известно, один из ключевых принципов многолетней стратегии Microsoft при работе на ИТ-рынке заключается в ориентации на массового пользователя, создании продуктов и технологий, полезных и доступных самому широкому кругу сотрудников организаций. Это очень хорошо видно и в сфере бизнес-аналитики, которая до недавнего времени предназначалась фактически только профессионалам в этой области и руководству компаний. Снизить планку доступности BI-средств удается за счет нескольких факторов, в числе которых:

  • уменьшение стоимости продуктов;
  • упрощение работы с ними (чтобы сделать их понятными не только опытным, но и рядовым пользователям);
  • расширение возможностей индивидуальной настройки и поддержки коллективной работы;
  • снижение затрат на сопровождение ИТ.

В общем виде подход Microsoft к решению этих задач опирается на максимальное использование существующей и знакомой (как пользователям, так и ИТ-специалистам) инфраструктуры исходя из того, что расширение клиентской базы в свою очередь позволяет понизить ценовую планку ПО и упростить его внедрение и эксплуатацию.

Основные возможности PerformancePoint Server 2007

В PPS 2007 входит три основных функциональных набора, необходимых для управления эффективностью бизнеса: контроль, анализ и планирование, благодаря чему организации могут согласовывать свою текущую деятельность со стратегическими целями и задачами. Каждый набор реализован в виде отдельного компонента.

PerformancePoint Monitoring представляет собой развитие приложения Business Scorecard Manager 2005. С его помощью можно определять и применять карты показателей (scorecards), информационные панели и ключевые параметры производительности (key performance indicators, KPI), что повышает уровень подотчетности и согласованности действий в рамках всей организации.

PerformancePoint Analytics — новая версия ProClarity Analytics. Этот компонент упрощает анализ и выявление тенденций, возможностей и угроз, а также позволяет собирать информацию и обмениваться ею в масштабе всего предприятия.

PerformancePoint Planning включает в себя набор новых функций, упрощающих планирование, составление бюджетов и прогнозирование.

Для оценки потенциала PPS 2007 имеет смысл рассматривать его возможности в различных аспектах. Так, с точки зрения бизнеса он имеет следующие преимущества:

  • возможность для бизнес-пользователей определять, изменять и поддерживать логику и бизнес-правила;
  • гибкая система на основе моделей;
  • поддержка процесса управления производительностью;
  • единая система для всего предприятия;
  • наглядность и прозрачность;
  • возможности аудита и высокий уровень безопасности.

В плане функциональности PPS 2007 предоставляет следующие средства:

  • планирование, бюджет, прогнозы, показатели, таблицы, отчеты и консолидация;
  • шаблоны и формы на основе рабочих процессов;
  • готовые и настраиваемые правила и расчеты;
  • единая модель данных и определений.

Для пользователей продукт обеспечивает следующие возможности:

  • вовлечение всех пользователей в процесс управления производительностью;
  • индивидуальная настройка показателей и планов;
  • мощные средства наглядного представления и заметок;
  • поддержка совместной работы;
  • уведомления и оповещения;
  • знакомый интерфейс офисных приложений, в том числе Excel.

В целом PerformancePoint Server предоставляет такие преимущества, как самообслуживание бизнес-пользователей, централизованное управление приложениями, возможность интеграции источников данных. Кроме того, решение по цене доступно для широкого развертывания и повышает отдачу от уже сделанных вложений в Microsoft Office и SQL.

Далее мы посмотрим, как потенциальные возможности PPS 2007 используются на различных этапах работы с ним.

Отслеживание (мониторинг)

Бизнес-пользователи могут создавать сводки показателей производительности благодаря встроенной поддержке отслеживания и анализа. Все сотрудники организации могут отслеживать ключевые показатели деятельности предприятия, что позволяет им не только принимать конкретные решения в рамках своих обязанностей, но и чувствовать "дыхание" всей компании (рис. 2). Все пользователи действуют в знакомой среде с поддержкой совместной работы, индивидуально настраивая представления в среде различных приложений (Office, SharePoint, Reporting Services).

Рис. 2. Мониторинг ключевых бизнес-показателей в PerformancePoint Monitoring.

Компонент PerformancePoint Monitoring имеет сертификат Balanced Scorecard Collaborative и позволяет работать с различными данными — от индивидуальных показателей производительности до формальных методологических сводок и каскадных сводок. Web-интерфейс и функции автономной работы предоставляют безопасный доступ к показателям для внутренних и внешних пользователей. Контекстно зависимые ключевые индикаторы производительности и отчеты отражают изменения в планировании, бюджете и прогнозах в реальном времени. Продукт обеспечивает быстрое и экономичное развертывание и интеграцию с приложениями Office. Встроенные шаблоны и мастера позволяют быстро создавать показатели и обмениваться ими. Доступ к широкому набору структурированных и неструктурированных источников данных обеспечивает удобство и высокую эффективность работы с показателями.

Анализ

Бизнес-пользователи обмениваются лучшими методиками бизнеса с помощью интуитивно понятной среды для совместной работы. Для рядовых пользователей расширенные функции наглядного представления и анализа облегчают понимание сложной информации, позволяют выявлять тенденции и важные моменты. Мощные средства анализа доступны всем сотрудникам и применяются без дополнительной настройки (рис. 3).

Рис. 3. Анализ структуры продаж с помощью PerformancePoint Analytics.

Анализ можно выполнять через Web-интерфейс и в автономном режиме, при этом гарантируется безопасный доступ к информации нужным пользователям в нужное время. Средства анализа включают приложения Office, а также расширенные возможности: деревья декомпозиции, детализация в строках, определения правил бизнес-аналитики, анализ причин и следствий и прогнозирование. Пользователи могут создавать аналитики, обмениваться и управлять ими без написания программного кода. Можно создавать и применять локальные и серверные вычисления, действия, заметки, таблицы показателей, а также ключевые индикаторы производительности, планирования, бюджетов и прогнозов. Обмен и управление аналитическими представлениями выполняются в SQL Server Report Manager или SharePoint Server Report Center; результаты можно сохранить в форматах PDF, HTML, XML.

Планирование

Бизнес-пользователи могут наиболее удобным для себя образом определять, изменять и поддерживать логические бизнес-модели, интегрированные с правилами, рабочими процессами и корпоративными данными. Остальные пользователи, выполняющие обработку информации, могут участвовать в бизнес-процессах планирования, проработки бюджета и прогнозирования в удобной и привычной среде Excel.

Создание моделей, интеграция и поддержка бизнес-правил, выполнение вычислений и описание предположений не требуют написания программного кода. Можно развернуть как единичную модель, так и корпоративные сценарии с сопоставлением ряда моделей. Доступны функции восходящего и нисходящего планирования, где поддерживается указание пользователями основных целей, тестовые сценарии и подробный анализ в оперативном и в автономном режиме. Управление процессами позволяет управлять ходом прогнозирования посредством форм, рабочих процессов, подачи заявок, подтверждений, отчетов, оповещений и примечаний. Имеется возможность подключения информации из внешних систем и источников данных. Правила преобразования и бизнес-логики можно применять как к PerformancePoint, так и к внешним системам и источникам данных.

Отчеты и консолидация

Данная функциональность не включена непосредственно в продукт PPS 2007, она реализуется на базовом уровне BI-платформы Microsoft (SQL Server + Excel). Тем не менее о ней нельзя не упомянуть, так как она фактически входит в "обязательную программу" при решении широкого круга задач бизнес-аналитики.

С помощью средств отчетности бизнес-пользователи могут определять собственное представление бизнеса и своевременно распространять шаблоны и формы. Все сотрудники могут объединить рабочие и финансовые данные в один отчет, не тратя времени на объединение данных вручную, при котором к тому же возможны ошибки. Пользователи, занимающиеся обработкой информации, могут создавать, настраивать отчеты о качестве производства и обмениваться ими в Excel, подключившись к безопасному серверу с централизованным управлением.

Функции финансового анализа дают бизнес-пользователям возможность выполнять анализ влияния и вариационный анализ (совокупность цена/количество/заказчик/продукт и сроки). Управление процессом финансовой консолидации поддерживает преобразование в различные денежные единицы, изменение структуры компании, многоуровневые процессы; имеется также поддержка GAAP и IFRS. Широко распространенные динамические и стандартные отчеты включают финансовые сводки, управленческие отчеты и произвольный анализ. Фильтры отчетов (переданные из Excel в Reporting Services) обеспечивают однородность отчетов для потребителей. Отчетами можно управлять с помощью SQL Server Report Manager и SharePoint Server Report Center.

Что такое Business Performance Management

Вот как определяется понятие BPM в отчете «Понимание управления производительностью для более эффективного управления бизнесом» (Gartner, октябрь 2006 г.): «Business Performance Management — это сочетание методологий и метрик управления, а также информационных технологий (приложений, средств и инфраструктуры), с помощью которого пользователи могут определять, отслеживать и оптимизировать результаты для достижения индивидуальных целей и решения задач отделов, действуя в рамках стратегических целей на нескольких организационных уровнях (индивидуальный уровень, уровень процессов, групп, отделов, компаний и экосистемы бизнеса)». В том же отчете говорится о том, что BPM связано с переходом от оптимизации решения частных и тактических задач к выработке эффективного комплекса стратегий компании, который представляет собой постоянный циклический процесс из трех сменяющих друг друга этапов: отслеживание (что произошло? что происходит сейчас?), анализ (почему?), планирование (что произойдет в будущем? чего желательно достичь?).

Тематика Business Performance Management была в центре внимания прошедшей в конце мая в Стокгольме конференции компании SAS Institute (http://www.sas.com) — одного из лидеров данного сегмента, стоявшего у его истоков. Еще лет десять назад компания начала формировать систему горизонтальных и вертикальных решений, объединив ее в какой-то момент в комплекс SAS for Performance Management, реализованный на базе SAS Enterpise Intelligence Platform. В одном из пленарных докладов на конференции старший вице-президент SAS Джим Дэвис представил любопытную общую схему повышения эффективности бизнеса через анализ деятельности предприятия, уверив при этом слушателей, что продукты SAS могут им помочь преодолеть все десять ступеней на пути к построению оптимизированной модели управления компанией (рис. 4).

Рис. 4. Путь к повышению эффективности бизнеса лежит через аналитику (источник: SAS Forum, 2007).

Перспективы развития

Несмотря на планы выпуска интегрированного продукта PerformancePoint Server 2007 до конца этого года (компоненты Monitoring и Analytics были доступны уже летом), вполне вероятно, что ProClarity Analytics Server будет продолжать развиваться и как самостоятельный продукт (в марте Microsoft выпустила PAS 6.3 уже под собственной маркой). И все же магистральный путь в области управления эффективностью бизнеса — это развитие PPS, который корпорация в 2008—2010 гг. обещает пополнить функциями, реализованными сегодня в ее ERP-продуктах (в частности, Dynamics Forecaster и Dynamics FRx).

Мнение эксперта
Системы управления эффективностью бизнеса

Николай Дорогов,
ведущий консультант практики информационно-аналитических систем компании Columbus IT Russia

Сравнительно недавно придя на мировой рынок BI-решений, корпорация Microsoft добилась серьезных успехов и признания. Развивая свое решение путем наращивания функциональности Microsoft SQL Server и используя популярность этой СУБД, компания обеспечила продвижение своего OLAP-инструмента Analysis Services, а позже и системы отчетности Reporting Services, сделав их естественной точкой входа в область BI для многих ИТ-специалистов. В то же время стало очевидно, что BI-системы следует рассматривать как часть комплексных систем управления эффективностью бизнеса.

Системы управления эффективностью должны содержать средства отчетности и анализа, планирования, бюджетирования и прогнозирования, а также средства измерения и мониторинга параметров эффективности с помощью специализированных информационных панелей и ключевых показателей деятельности, например, Balanced Scorecards. При этом важен такой фактор, как степень интеграции указанных средств и наличие специализированной методологии внедрения систем управления эффективностью.

Компания Columbus IT Russia занимается внедрением систем управления эффективностью бизнеса с 2003 г. Наш стратегический партнер по поставке решений данного класса — компания Cognos (http://www.cognos.com). Не пытаясь действовать во всех сегментах ИТ-рынка, Cognos сосредоточила усилия именно на развитии систем управления эффективностью. В настоящее время платформа Cognos объединяет основные современные концепции в данной области — планирование деятельности компании (Enterprise Planning), бизнес-аналитику и систему сбалансированных показателей. При этом каждый из элементов платформы входит в число мировых лидеров в своем классе. Следует также отметить глубокую интеграцию элементов системы в рамках единой платформы. Для пользователей это означает единую точку доступа ко всем задачам управления эффективностью. Широкое использование Web-технологий делает систему более гибкой и позволяет сократить затраты на внедрение и поддержку. Например, не только просмотр, но и разработка новых форм отчетов, внесение плановых показателей, интерактивный анализ данных выполняются через браузер.

Проекты внедрения систем управления эффективностью имеют свои особенности по сравнению с системами других классов, в частности, ERP. Чтобы добиться высокого качества выполнения проектов, необходима методология, учитывающая эти особенности. Используемый нами подход основан на разработанной Cognos методологии SIM (Solutions Implementation Methodology), аккумулирующей огромный опыт внедрений компании Cognos и ее партнеров по всему миру.

Таким образом, для качественного внедрения системы управления эффективностью недостаточно иметь просто набор инструментов для решения разрозненных задач. Необходимо также обеспечить глубокую интеграцию элементов системы, разработать серьезную методологическую основу для ведения проектов, привлечь специалистов, имеющих как опыт работы с конкретными программными продуктами, так и глубокое понимание предметной области.

Усилия Microsoft по развитию систем управления эффективностью бизнеса показывают высокую заинтересованность компании в данном направлении, что позволяет ей рассчитывать в перспективе занять серьезное положение среди лидеров данного сегмента. Однако эта задача требует значительного времени и больших усилий при конкуренции с компаниями-лидерами, имеющими комплексные решения и разработанные методологии в данной области.
